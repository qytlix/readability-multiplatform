## Issue #23 — 恢复稳定渲染基线并重构前端布局

| 字段 | 内容 |
|---|---|
| **状态** | OPEN |
| **作者** | @chaocyndrome (Chen Yinuo) |
| **标签** | `enhancement`, `frontend` |
| **负责人** | @chaocyndrome (Chen Yinuo) |
| **父 Issue** | #24 |

### 背景

Windows 高 DPI 环境下存在页面比例和三栏布局异常。此前使用全局缩放补偿会导致界面跳变和整体比例失衡，需要先恢复稳定的渲染基线，再整理 Renderer 布局代码。

### 工作范围

1. 排查并移除异常或临时的全局缩放补偿；
2. 恢复稳定、统一的页面渲染比例；
3. 整理三栏布局相关代码，明确布局计算、状态管理、组件结构和 CSS 的职责；
4. 根据真实容器宽度调整 **Feed、Entry 和 Reader** 的空间分配；
5. 验证窗口缩放、最大化、折叠和拖拽等现有行为。

### 协作边界

- 只修改 **Renderer 的界面与布局代码**，以及恢复渲染基线直接需要的窗口缩放设置；
- **不修改** Feed/Sync 后端、SQLite、AI Provider、数据契约和普通 IPC 逻辑；
- 如果发现必须调整公共接口、Main 或 Preload，先与总体重构负责人（#24）沟通；
- 总体重构期间，Renderer 布局相关文件由本 Issue 统一修改，避免双方并行改动同一部分。

### 验收标准

- [ ] 不再依赖按分辨率硬编码的全局缩放；
- [ ] 三栏布局能够根据真实可用宽度正常调整；
- [ ] macOS 原有显示不退化；
- [ ] 完成 Windows 150% 缩放环境复测；
- [ ] 现有功能、测试和构建检查保持正常。
