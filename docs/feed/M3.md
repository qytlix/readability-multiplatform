## 3. 任务与拆分

### M3-FEED-01：清洗流水线版本化缓存（1d）

#### 范围内

实现按版本号按层重建和缓存失效机制（参考 IMPLEMENTATION_PLAN.md §10.2）：

```
1. Render cache 与版本匹配 → 直接返回（由组长 Reader 模块处理）
2. Markdown 有效但 render 过期 → 只重渲染（Reader 模块）
3. Cleaned HTML 有效但 Markdown 过期 → 重建 Markdown
4. Source HTML 有效但 Readability 过期 → 重跑清洗
5. 上游数据缺失 → 重新联网获取
```

**本模块负责：layer 3~5 的缓存决策逻辑**

```typescript
class ContentCacheManager {
  // 判断是否需要重建某一层
  needsRebuild(entryId: number, targetLayer: CacheLayer): Promise<RebuildDecision>
  
  // 按需重建到指定层
  async rebuildTo(entryId: number, targetLayer: CacheLayer): Promise<CleanedContent>
}
```

**关键实现**：

- `readabilityVersion`：每次更新 Readability 库或清洗规则时递增
- `markdownVersion`：每次更新 Markdown 转换器规则时递增
- 两个版本号存储在 `entry_content` 表中，与当前应用版本比较
- `sourceContentHash`：基于 source HTML 的 SHA-256，用于检测源内容是否变化

**缓存失效规则**：

| 当前状态 | 应用版本 | 决策 |
|---|---|---|
| cleanedHtml 存在 | readabilityVersion 匹配 | 层 3 有效 |
| cleanedHtml 存在 | readabilityVersion 不匹配 | 层 4 → 重跑清洗 |
| cleanedMarkdown 存在 | markdownVersion 匹配 | 层 2 有效 |
| cleanedMarkdown 存在 | markdownVersion 不匹配 | 层 3 → 重建 Markdown |
| sourceHtml 存在 | 不需要重建清洗 | 层 4 有效（联网跳过） |
| 所有数据缺失 | — | 层 5 → 重新联网 |

**并发保护**：
- 对同一 Entry，同一时刻只允许一个 rebuild 操作
- 使用 entry-level build lock（内存 Map 或数据库标记）

#### 验收标准

- [ ] readabilityVersion 变化时自动重跑清洗
- [ ] markdownVersion 变化时只重建 Markdown（不重新联网）
- [ ] 版本未变时直接返回已有内容
- [ ] sourceContentHash 变化时重新联网获取
- [ ] 并发重复请求不导致多次重建
- [ ] 重启后版本信息正确读取

### M3-FEED-02：文章搜索（标题 + 摘要） [P1，按容量选择]（0.5d）

#### 范围内

```typescript
interface SearchQuery {
  query: string;                    // 搜索关键词
  feedId?: number;                  // 可选：限定 Feed
  limit: number;                    // 每页数量
  cursor?: { publishedAt: string; id: number };  // keyset 分页
}

interface SearchResult {
  entries: EntryListItem[];
  nextCursor?: { publishedAt: string; id: number };
  totalMatches: number;
}
```

**实现方式**：
- SQLite `LIKE` 查询：`WHERE title LIKE '%query%' OR summary LIKE '%query%'`
- 300ms debounce 在 Renderer 侧
- keyset 分页（基于 publishedAt + id 排序后过滤）
- 旧查询响应不覆盖新查询结果（请求取消或响应丢弃）

**IPC/Preload**：
- `entry:search` 通道
- Preload `entry.search(query, params)`

#### 范围外
- 全文搜索（cleanedMarkdown 内搜索）— Stretch
- 模糊匹配、拼音搜索 — Stretch
- 搜索索引 — Stretch

#### 验收标准

- [ ] 输入关键词后匹配标题的文章出现在结果中
- [ ] 输入关键词后匹配摘要的文章出现在结果中
- [ ] 切换查询后旧结果不闪现
- [ ] keyset 分页正常工作
