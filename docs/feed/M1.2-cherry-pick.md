# M1.2 — Cherry-pick: `cyn/feat-renderer` → `qyt/feed`

## 背景

将 `cyn/feat-renderer` 分支中的 renderer 进度合并到 `qyt/feed` 分支。

- **源分支：** `cyn/feat-renderer`（commit `ecf9981..d7715b4`，共 7 个）
- **目标分支：** `qyt/feed`
- **共同起点：** `f1f10e6`（`docs: 添加 M1 修复总结文档`）
- **执行日期：** 2026-07-15

## 策略

采用**方案 A**（cherry-pick）而非方案 B（merge），原因：

1. `ecf9981`（"implemented feed in windows"）只改了 `forge.config.ts` 和 `package-lock.json`，而 `qyt/feed` 上这两个文件已包含更完善的改动（executableName、Squirrel 配置、maker 选项、OnlyLoadAppFromAsar 等），无需覆盖。
2. 跳过 `ecf9981`，只 cherry-pick 纯 renderer 的 5 个 commit（`e72e0dc..d7715b4`），避免 `forge.config.ts` 和 `package-lock.json` 的冲突。
3. 源分支 `cyn/feat-renderer` 在 cherry-pick 后完整保留。

## Cherry-pick 过程

```bash
git cherry-pick e72e0dc..d7715b4
```

### 冲突与解决方案

所有冲突均出现在 `src/renderer/index.css`，冲突原因：`qyt/feed` 的基线版本（`f1f10e6`）与 `cyn/feat-renderer` 的 CSS 改动不一致，导致 cherry-pick 每个 commit 时增量 patch 无法干净应用。

| Cherry-pick commit | 冲突位置 | 解决方案 |
|---|---|---|
| `fa6248b` (fix list) | `.feed-item` — `width: 100%` 重复 | 合并：保留 renderer 的 `min-height: 36px`，去掉重复 `width: 100%`；`.feed-item-error` 和 `.feed-item-count` 采用 theirs（renderer 版本） |
| `7505eea` (fix Sync bottom) | `.btn-icon` vs `.sync-button`/`.sync-icon` | renderer 重构了 sync 按钮样式，采用 theirs |
| `ab3addf` (enable adjusting folder) | `:root` 色系变量（`--bg-*` vs `--slate-*`）、`.app-body`（flex vs grid）、`.app-sidebar`/`.app-main` | renderer 改为 `slate-*` 色系和 grid 布局，采用 theirs |
